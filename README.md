## 实现方案

### 架构

```
单文件 HTML（CSS + JS + 数据全部内联）
         │
         ├─ 数据层：78 张牌的完整解读数据（CARDS + MINOR_ARCANA）
         ├─ 解读引擎：bigSummary() 模板引擎 + AI API 实时生成
         ├─ 交互层：分批翻牌 / 弹窗总览 / 横排滑动 / 长按详情
         └─ 部署：nginx 静态文件 + VPS 图片托管 + Node.js API 端点
```

### 数据设计

每张卡牌的数据结构：
```js
{
  id, name, en, kw, el, img, persona,  // 基础信息
  up:  { core, q, what, warn },         // 正位解读（释义 / 问题相关 / 行动建议 / 注意事项）
  rev: { core, q, what, warn }          // 逆位解读
}
```

- `core`：牌义核心，无问题时展示
- `q`：针对用户问题的个性化回答
- `what`：行动建议（长按弹窗显示）
- `warn`：注意事项（长按弹窗显示）

### 解读引擎

两层解读架构：

- **第一层（AI 实时解读）**：`showReading()` → 打包牌阵数据 → POST 到 VPS API → DeepSeek 返回一段塔罗师口吻的连贯叙事
- **第二层（本地降级）**：`bigSummary()` 模板引擎，AI 不可用时自动回退

### 元素系统

ELEMENT_TONE 词库为火/风/水/土四元素各定义一套语言风格：

| 元素 | 形容词 | 名词 | 动词 | 节奏 |
|------|--------|------|------|------|
| 火 | 热烈的 | 行动力 | 推动 | 迅猛的 |
| 风 | 清晰的 | 思维力 | 展开 | 灵动的 |
| 水 | 深沉的 | 感受力 | 渗透 | 缓慢而深入的 |
| 土 | 踏实的 | 实践力 | 沉淀 | 步步为营的 |

解读正文的措辞会随抽到的牌的元素底色动态变化。

### 翻牌 UI

- ≤3 张牌：一屏全展示，一张一张翻
- 5 张牌：3+2 两批翻
- 10 张牌（凯尔特十字）：3+3+4 三批翻
- 全部翻完 → 弹窗网格总览 → 点击「查看解读」→ 横排滑动浏览全部牌面 + AI 叙事

## 项目亮点

1. **零依赖**：纯 HTML + CSS + JS，无框架，无构建工具，一个文件跑起来
2. **AI 原生**：解读不是模板填词，而是每次实时生成独一无二的塔罗师叙事
3. **元素渗透**：不是给文字加标签，而是让元素的底色改变措辞本身——火用短句和动词，水用长句和意象
4. **降级优雅**：AI API 不可用时自动回退本地模板，不影响用户体验
5. **78 张全牌库**：从零手写每张牌的 8 段解读文字，AI 生成全 Art Nouveau 风格卡面
6. **细节打磨**：3D 翻转动画、scroll-snap 吸附滚动、content-visibility 性能优化、分批翻牌仪式感
